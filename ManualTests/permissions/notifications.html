Notifications

Browser UI Expectations

  • Notification.requestPermission() shows the URL-bar authorization prompt when site/system state allows prompting.
  • Dismiss resolves the promise to default and must not store a site Block.
  • Repeated dismissals during cooldown also resolve default, not persistent denied.
  • macOS notification denial shows system-blocked UI. Site denial and macOS notification authorization remain separate.
  • URL hub -> Permissions and Privacy -> Site Settings show saved Notifications decisions when present.

Controls

No icon is sent because this manual suite has no local bitmap icon asset.
