Geolocation

Expected: first-time location requests open the URL-bar anchored Sumi permission prompt. Allow this time grants only the current page/request lifetime, allow while visiting persists for a normal profile, don't allow persists for a normal profile, and dismiss does not persist. If macOS Location Services blocks access, the prompt should show a system-blocked explanation and System Settings action.

URL hub check: open Site Controls from the URL bar, open Permissions, and confirm Location shows the current site decision, active status during a watch, and macOS Location Services blocked state separately from site Block. Changing Ask, Allow, or Block should affect future requests only.

Runtime controls check: during an active provider session or watch, the URL-bar active icon should open Permissions and show Active now controls. Pause location sharing should stop new coordinate delivery without changing the stored site decision. Resume location sharing should work only while the page session is still valid and the site permission remains allowed. Stop sharing location this visit should stop delivery for this page visit without persisting a site deny.
