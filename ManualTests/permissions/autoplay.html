Autoplay

Browser UI Expectations

  • Autoplay is a site setting, not a permission prompt.
  • URL hub -> Permissions controls Autoplay with Default, Allow all autoplay, Block audible autoplay, and Block all autoplay when supported.
  • Privacy -> Site Settings controls saved Autoplay exceptions.
  • Active pages may show Reload to apply autoplay setting because Sumi applies policy through WebKit configuration.
  • The old UserDefaults autoplay store is not used.

Controls

This page intentionally attempts autoplay on load and when Retry is clicked. Media is generated locally with canvas and Web Audio; no media files or network URLs are used.

Audible video autoplay

Not attempted.

Muted video autoplay

Not attempted.

Audio element autoplay

Not attempted.

Manual Checklist

      
    
        
  1. With browser default, record audible video, muted video, and audio element results.
    2. 
        
  2. Set URL hub -> Permissions -> Autoplay to Allow all autoplay, reload if prompted, and retry.
    3. 
        
  3. Set Autoplay to Block audible autoplay, reload if prompted, and confirm audible media is blocked while muted media follows WebKit behavior.
    4. 
        
  4. Set Block all autoplay if the Sumi row exposes it, reload if prompted, and confirm all automatic playback is blocked.
    5. 
        
  5. Use Play all with user gesture; playback should succeed when the site has a fresh user gesture.
    6. 
        
  6. Reset to Default in URL hub and Privacy Site Settings.
