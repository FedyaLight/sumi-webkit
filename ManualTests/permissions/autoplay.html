Sumi Autoplay Permission Test

Manual expectations

  • Default should match current Sumi behavior.
  • Allow should permit configured autoplay after reload or rebuild if required.
  • Block should prevent audible autoplay after reload or rebuild if required.
  • Muted video autoplay follows WebKit's normal muted-media behavior.
  • The URL hub Permissions submenu Autoplay row should write to the canonical permission store, not old UserDefaults values.
  • The top-level Site Controls popover should route through Permissions for autoplay and show reload or rebuild required when the active page needs it.
  • When an active-page autoplay change requires reload or rebuild, Permissions should show Reload to apply autoplay setting. Clicking it should use the existing tab reload/rebuild path, clear only after reload or matching policy state, and never show an autoplay prompt.
