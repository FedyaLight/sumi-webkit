Media Permission Manual Test

Expected: first-time camera, microphone, or combined requests open the URL-bar anchored Sumi permission prompt. Allow this time should grant again only within the same current page visit, for the same exact site/profile/type/page id. Requesting the same device again on this page should reuse the one-time grant; reload, cross-document navigation, tab close, or WebView replacement should ask again. Allow while visiting should persist for a normal profile, don't allow should persist for a normal profile, and dismiss should not persist.

URL hub check: open Site Controls from the URL bar, open Permissions, and confirm Camera and Microphone show the current site decisions plus active or muted status after a stream starts. Changing Ask, Allow, or Block should affect future requests only, and Reset permissions should not clear cookies, site data, or tracking-protection overrides.

Runtime controls check: after allowing microphone or camera, the URL-bar active icon should remain visible while capture is active. Clicking the active icon should open Permissions and show an Active now section. Mute and Unmute should change WebKit capture state where supported; Stop should end capture where supported or show a clear unsupported/no-op result. Stored Ask, Allow, or Block settings should not change.

Anti-abuse check: dismiss the same camera or microphone prompt repeatedly. Immediate re-prompts should pause for the same site/profile/device key, but Site Controls and Site Settings should not show a stored Block. Manually setting Allow or resetting the exact permission should clear that pause.

Automatic cleanup check: with cleanup enabled in Privacy Settings, stale saved Allow decisions may be removed when the cleanup service runs. Active capture should not be interrupted; the next request or navigation should observe the removed saved Allow.
