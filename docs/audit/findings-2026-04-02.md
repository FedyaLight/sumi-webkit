# Sumi Audit Findings — 2026-04-02

## P1

### 1. The scheme advertises UI tests, but the repository has no actual `SumiUITests` sources

- Evidence: `Sumi.xcodeproj/xcshareddata/xcschemes/Sumi.xcscheme`
- Impact: high-value regressions in launcher lifecycle, floating URL bar, Glance, split and space switching are effectively unguarded.
- Parity status: partial
- Owner area: test infrastructure

### 2. Theme token flow still leaks through direct gradient-brightness checks

- Evidence:
  - `Navigation/Sidebar/SpacesList/SpacesListItem.swift`
  - `Navigation/Sidebar/SpacesList/SpacesList.swift`
- Impact: some chrome surfaces still derive color directly from `windowState.gradient.primaryColor.isPerceivedDark`, which can diverge from Zen token-driven theming and produce inconsistent icon/text contrast during theme changes.
- Parity status: partial
- Owner area: theme and chrome

### 3. Space restore still falls back to creating a Google tab instead of staying empty

- Evidence: `Sumi/Managers/TabManager/TabManager.swift`
- Impact: this conflicts with Sumi's current empty-state contract and can mask close/restore regressions by silently creating a live page.
- Parity status: partial
- Owner area: tabs and session restore

### 4. Launcher and legacy pinned paths still coexist

- Evidence:
  - `Sumi/Managers/TabManager/TabManager.swift`
  - `Sumi/Managers/BrowserManager/BrowserManager.swift`
  - `Navigation/Sidebar/SpacesList/SpacesListItem.swift`
  - `Navigation/Sidebar/SpaceContextMenu.swift`
- Impact: counts, selection, display and delete dialogs still read from both `spacePinnedTabs(...)` and `spacePinnedPins(...)`. This can drift between live runtime state and launcher-only state.
- Parity status: partial
- Owner area: launcher lifecycle / DnD / counts

## P2

### 5. Space creation can still hard-crash if no profile exists

- Evidence: `Sumi/Managers/TabManager/TabManager.swift`
- Impact: migration or corrupted-profile edge cases can terminate the app instead of recovering.
- Parity status: partial
- Owner area: profile and space bootstrap

### 6. Sidebar context menus still ship disabled TODO actions in user-facing paths

- Evidence:
  - `Navigation/Sidebar/SpaceContextMenu.swift` (`Duplicate Space`)
  - `Sumi/Components/Sidebar/SpaceSection/SpaceTab.swift` (`Share`, `Close Others`)
- Impact: visible parity gap with Zen and signal that some workflows are not actually finished.
- Parity status: missing
- Owner area: sidebar UX polish and command completeness

### 7. Core managers still contain heavy runtime debug logging in hot paths

- Evidence:
  - `Sumi/Managers/TabManager/TabManager.swift`
  - `Sumi/Components/WebsiteView/WebsiteView.swift`
  - `Sumi/Managers/WebViewCoordinator/WebViewCoordinator.swift`
- Impact: logs increase noise during live debugging, make true failures harder to isolate, and suggest some code paths are still being stabilized manually.
- Parity status: partial
- Owner area: platform stability / observability

## P3

### 8. Missing Zen parity backlog is still substantial

- Evidence: Zen test suites under `src/zen/tests/{folders,glance,spaces,split_view,tabs,urlbar,window_sync}`
- Missing or incomplete areas:
  - folder owner/live-folder behavior
  - workspace bookmarks and double-click new-tab semantics
  - full window-sync parity
  - complete split-with-folder and split-with-glance coverage
- Impact: these are not necessarily bugs in current Sumi features, but they remain clear gaps versus the chosen Zen baseline.

## Recommended Next Batches

1. Stabilize the core state machine:
  - unify launcher selection, display and count paths
  - remove default-Google fallback from restore and empty-state flows
  - eliminate crash-only branches in space/profile bootstrap
2. Build repeatable regression coverage:
  - use the new accessibility identifiers as hooks
  - start with live smoke and high-value UI automation targets
3. Finish theme token cleanup:
  - remove remaining gradient-brightness branches from chrome surfaces
  - verify contrast on all transient surfaces
4. Work through missing Zen parity backlog domain by domain:
  - folders
  - spaces
  - split and Glance
  - window sync
